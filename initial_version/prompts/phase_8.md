# Phase 8: 프로젝트 학습 가이드 생성

Phase 1~7로 프로젝트가 완성된 후, 이 프롬프트를 실행하여 `MATERIAL.md` 학습 가이드를 생성합니다.

---

## 프롬프트

```
# 역할

시니어 개발자로서 현재 프로젝트의 학습 가이드 `MATERIAL.md`를 작성해주세요.
이 문서를 읽은 주니어 개발자가 프로젝트를 이해하고 유지보수할 수 있어야 합니다.

---

# 작업

## 1. 프롬프트 파일 읽기

**중요**: Phase를 임의로 나누지 마세요!

`prompts/phase_1.md` ~ `prompts/phase_7.md` 파일을 순서대로 읽으세요.
이 프롬프트들이 프로젝트를 만들 때 실제로 사용된 것이며, 각 프롬프트가 하나의 Phase입니다.

| 프롬프트 파일 | Phase |
|--------------|-------|
| prompts/phase_1.md | Phase 1 |
| prompts/phase_2.md | Phase 2 |
| ... | ... |
| prompts/phase_7.md | Phase 7 |

## 2. 프롬프트 → 코드 매핑

각 프롬프트 파일을 읽고, 그 프롬프트로 생성된 코드 파일들을 파악하세요:

- 프롬프트에 "생성할 파일" 또는 구현 요구사항이 명시되어 있음
- 해당 파일들의 실제 구현 코드를 읽고 분석

## 3. MATERIAL.md 작성

아래 형식으로 작성:

---

# [프로젝트명] 학습 가이드

이 문서는 프로젝트를 7단계 프롬프트 순서에 따라 이해하기 위한 학습 가이드입니다.
각 Phase별로 **프롬프트 역할**, **핵심 개념**, **주요 코드**, **데이터 흐름**을 설명합니다.

---

# 목차

1. [Phase 1: (phase_1.md의 제목)](#phase-1-제목)
2. [Phase 2: (phase_2.md의 제목)](#phase-2-제목)
...
7. [Phase 7: (phase_7.md의 제목)](#phase-7-제목)
8. [전체 데이터 흐름 다이어그램](#전체-데이터-흐름-다이어그램)

# 작성 규칙

1. **Phase 구분**: 반드시 `prompts/phase_N.md` 프롬프트 파일 단위로 Phase를 구분 (임의로 나누지 말 것!)
2. **프롬프트 참조**: 각 Phase 설명 시 해당 프롬프트 파일의 내용을 근거로 작성
3. **코드 인용**: 반드시 `파일경로:라인번호` 포함, 핵심 부분만 발췌
4. **다이어그램**: ASCII 박스와 화살표 사용 (`┌─┐ → ▼`)
5. **설명**: "왜" 이렇게 했는지 포함, 한국어로 작성
6. **누락 없이**: 프롬프트에서 요구한 모든 파일과 함수 설명

1. `prompts/phase_1.md` 읽기 → 생성된 코드 파일 확인 → Phase 1 섹션 작성
2. `prompts/phase_2.md` 읽기 → 생성된 코드 파일 확인 → Phase 2 섹션 작성
3. ... (phase_7.md까지 반복)
4. 전체 데이터 흐름 다이어그램 작성
5. 학습 체크리스트 작성
